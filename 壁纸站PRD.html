一、产品概述

DNF 官方壁纸站是地下城与勇士（DNF）游戏官方的高清壁纸展示与分发平台，面向全平台用户提供：

  • 官方壁纸：版本KV主视觉、活动/节日限定壁纸等多分辨率下载
  • 同人作品：社区创作者投稿的优质同人作品展示与下载 + 公告/活动信息

2.5 壁纸列表（双布局模式）

布局模式一：官方壁纸（type='normal'）— 分组 Grid 布局

  • 每 5 张壁纸为一组，渲染为一个 .grid-row（display:grid; grid-template-columns:repeat(4,1fr); height:280px）
  • 每组第 1 张为大图（占 2 列 × 2 行），第 2~5 张为小图（各占 1 格）
  • 大图左右交替：奇数组大图在左侧（grid-column 1~2），偶数组大图在右侧（grid-column 3~4），形成视觉节奏感
  • 不足 5 张时全部等大排列，4 列单行（高度 150px）
  • 每页 20 条（PAGE_SIZE=20），底部显示分页器（页码按钮 + 前后翻页 + 范围提示）

布局模式二：同人作品（type='ugc'）— CSS 瀑布流

  • column-count:4 多列瀑布流（平板 3 列，手机 2 列），column-gap:12px
  • 壁纸含 orient 字段区分横版(h，缩略图 400×225)和竖版(v，缩略图 400×700)，渲染时按"2横+1竖+1横"节奏交替插入，保证横竖图均匀分布
  • 竖版图 max-height:360px; object-fit:cover 防止过长
  • 不分页，全部展示。底部显示"海量作品"提示卡片（蓝紫渐变背景，内容"✨ 海量作品 持续更新中 + 共收录 XX 幅同人佳作"）

搜索结果布局

  • 按 Enter 触发全局搜索，匹配标题/日期/扩展字段值（大小写不敏感子串匹配）
  • 搜索结果按一级分类分组显示，每组有独立标题头（分类图标 + 分类名 + 结果计数标签）+ 底部分割线
  • 每组内统一使用 4 列等高 Grid 平铺（卡片高 160px），不分页
  • 清空搜索框按 Enter 或点击导航分类按钮退出搜索模式

2.6 公告 + 活动区（仅同人作品）

功能说明

  • 仅在同人作品分类下，投稿横幅和壁纸列表之间显示
  • 左右两栏并排：左栏公告、右栏活动
  • 每栏最多展示 4 条（按 sort 字段升序排列），每条有彩色圆点（公告蓝色/活动橙色）+ 标题文本 + 可选标签（NEW/HOT）+ 右侧日期
  • 右上角「更多 →」按钮：公告跳转 #/notice，活动跳转 #/activity
  • 点击每条公告/活动项跳转对应详情页（#/notice/{id} 或 #/activity/{id}）

Hash 路由跳转

路由页面
#/notice公告列表页（展示所有公告，卡片式列表）
#/notice/{id}公告详情页（标题 + 日期 + 标签 + 正文）
#/activity活动列表页
#/activity/{id}活动详情页
  • 详情页「← 返回」按钮使用 history.back() 导航，确保从列表页进入时能正确返回列表而非主页
  • 列表页「← 返回壁纸站」按钮使用 location.hash='' 返回主页
  • 路由切换时隐藏主页所有区块（Banner/Tab/投稿横幅/公告活动区/壁纸列表），显示路由页面容器（最大宽度 900px 居中）
  • 详情内容支持 HTML 富文本渲染（加粗、标题、列表、引用、图片等），兼容旧纯文本（自动将 \n 转为 <br>）

三、后台管理功能

后台供运营人员管理所有前台展示内容，分为六个管理模块。所有编辑操作通过弹窗表单进行，保存后即时生效。带 * 的为必填项。

3.1 后台整体框架

模块总览

  1. Banner 管理 — 首屏走马灯轮播图的增删改查、排序、上下线控制
  2. 分类管理 — 一级分类（官方壁纸/同人作品等）及其下属二级分类的管理
  3. 壁纸管理 — 所有壁纸内容的增删改查，包含扩展字段和多分辨率下载资源
  4. 公告管理 — 同人作品公告的增删改查，含富文本详情内容
  5. 活动管理 — 同人作品活动的增删改查，含富文本详情内容
  6. 投稿配置 — 同人作品投稿入口的邮箱和说明文案配置

通用能力

  • 每个模块均支持：新增、编辑、删除操作
  • 编辑通过弹窗表单进行，必填项用红色 * 标记
  • 操作完成后 Toast 提示反馈（如"已保存""已删除"）
  • 删除操作需二次确认弹窗

3.2 分类管理

一级分类

  • 可新增、编辑、删除一级分类
  • 编辑弹窗内容：分类名称*、类型*（普通/UGC投稿类）、跳转链接、排序*、启用状态*
  • 跳转链接：可选字段。留空时前台点击正常切换分类；填写 URL 后前台点击该分类将新窗口打开链接而非切换分类
  • 类型为「普通」的分类支持在弹窗内直接添加/管理二级子分类
  • 类型为「UGC投稿类」的分类，前台会自动显示投稿入口横幅
  • 删除一级分类时提示"关联壁纸将失去分类"的确认弹窗

二级分类

  • 在一级分类的编辑弹窗内嵌管理，支持动态增删行
  • 每个二级分类填写名称和跳转链接（可选），排序按添加顺序自动生成
  • 跳转链接：留空时前台点击 Tab 正常筛选壁纸；填写 URL 后前台点击该 Tab 将新窗口打开链接
  • 不配置二级分类时，壁纸直接挂在一级分类下

功能说明

  • 集中管理前台同人作品的投稿入口，配置项精简为三个核心内容
