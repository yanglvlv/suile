# 政策配置组件及配置项说明文档

> 基于 `policy_config_v2.html` V10.1 版本整理

---

## 一、整体架构概览

```
政策配置
├── 1. 活动基础信息（活动名称、活动周期）
├── 2. 数据导入 & 字段标准化映射
└── 3. 赛道画布（N 个赛道 Tab）
    ├── 赛道属性面板（右侧面板）
    │   ├── 赛道名称
    │   ├── 活动时间（可独立设定）
    │   ├── 合并关系（add / max / mutex）
    │   ├── 互斥组成员（mutex 时显示）
    │   └── 有效天数定义
    └── 算子节点画布（9 种算子，可拖拽排序）
        ├── ① 准入/筛选条件（condition-filter）
        ├── ② 计算字段定义（computed-field）
        ├── ③ 阶梯排名发奖（rank-rule）
        ├── ④ 阶梯达标/溢出即发（target-rule）
        ├── ⑤ 固定金额即发（fixed-reward）
        ├── ⑥ 固定奖池平分（pool-rule）
        ├── ⑦ 多周期奖池瓜分（multi-period-pool）
        ├── ⑧ 流水分成/返点（share-rule）
        ├── ⑨ 裂变伯乐激励（referral-rule）
        └── ⑩ 表格直发（direct-rule）
```

| 配置项 | 类型 | 说明 |
|--------|------|------|
| **活动名称** | 下拉选择 | 从预设的12个政策模板中选择（分"活跃政策"10项 + "库存政策"3项） |
| **开始时间** | 日期 | 活动整体开始日期，所有赛道的默认起始时间 |
| **结束时间** | 日期 | 活动整体结束日期，超过此日期的数据不计入结算 |

**预设活动列表：**
- 活跃政策：天龙八部手游、FC ONLINE(端游)、FC足球世界(手游)、TF直播激励、石器时代:觉醒、天堂2-随乐五部、MDNF官签主播、MDNF官签机构、DNF端游直播、随乐-星座共鸣S2
- 库存政策：终极角落(周结算)、DNF手游随乐版、DNF官方机构版

## 三、数据导入与字段标准化

| 配置项 | 说明 |
|--------|------|
| **文件上传** | 支持 `.xlsx` / `.csv` 格式的结算数据表 |
| **模拟数据** | 内置15条模拟数据用于快速体验 |
| **自动映射** | 系统自动将导入列名与标准字段库（FIELDS，80+个字段）比对 |
| **映射方式** | 精准命中 → 别名纠偏（相似度≥50%自动匹配） → 待确认（需手动选择） |
| **推荐候选** | 未匹配的列会推荐 Top3 最相似的标准字段，支持一键采纳 |
| **数据预览** | 展示前10行导入数据 |
| **异常检测** | 检测空值、类型错误（数值列含非数字）、负数等异常 |
| **统计摘要** | 展示数据行数、精准命中数、别名纠偏数、待确认数、异常行数 |

**标准字段库（部分）：** 场观UV、直播总时长(h)、有效天数、已执行总价、总互动量、平均在线人数ACU、CPM、影响力积分、付费金额、机构ID等80+个字段

## 四、赛道属性面板

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **赛道名称** | `name` | 文本 | 赛道标签页上显示的名称，建议格式：平台-奖励类型 |
| **活动时间** | `activityTimeStart`/`activityTimeEnd` | 日期 | 每个赛道可独立设定考核时间范围，留空则使用活动整体时间 |
| **合并关系** | `relation` | 下拉 | 决定该赛道与其他赛道金额的合并方式（见下表） |
| **互斥组成员** | `mutexGroup` | 多选 | 当合并关系为"互斥"时显示，勾选哪些赛道与当前赛道互斥 |
| **有效天数定义** | `validDayHours` | 数字 | N小时=1个有效天，留空/0使用全局默认(1小时=1天) |

### 合并关系（3种）

| 值 | 名称 | 说明 |
|-----|------|------|
| `add` | 叠加 | 各赛道金额累加（**默认**） |
| `max` | 取最高 | 多赛道取最高金额，只发最多的那个 |
| `mutex` | 互斥多选一 | 与指定赛道组互斥，主播只能选一个领取 |

**互斥组行为：** 勾选互斥成员后会**自动双向同步**，对方也会被标记为互斥关系。取消互斥后自动回退为 `add`。

## 五、通用条件组（核心复用组件）

多个算子（准入/筛选条件、排名每档准入条件、固定金额准入条件、多周期达标条件等）都使用同一套条件组组件。

### 条件组结构

| 属性 | 类型 | 说明 |
|------|------|------|
| `logic` | 枚举 | `AND`(全部满足) / `OR`(任一满足) / `NOT`(取反)，点击循环切换 |
| `conditions` | 数组 | 条件行列表，每行为简单条件或嵌套子条件组 |

### 简单条件行

| 属性 | 类型 | 说明 |
|------|------|------|
| `type` | `'simple'` | 标识为简单条件 |
| `field` | 下拉 | 从标准字段库中选择（80+字段）+ "自定义..." |
| `operator` | 下拉 | 比较运算符（8种，见下表） |
| `value` | 输入 | 比较值 |
| `value2` | 输入 | 仅BETWEEN时显示，作为区间上限 |

### 嵌套子条件组

| 属性 | 类型 | 说明 |
|------|------|------|
| `type` | `'group'` | 标识为嵌套组 |
| `logic` | 枚举 | 子组自己的逻辑运算符 |
| `conditions` | 数组 | 递归嵌套，支持**无限层级** |

### 比较运算符（8种）

| 运算符 | 标签 | 值类型 | UI表现 |
|--------|------|--------|--------|
| `>` | 大于 | 单值 | 单输入框 |
| `>=` | 大于等于 | 单值 | 单输入框 |
| `<` | 小于 | 单值 | 单输入框 |
| `<=` | 小于等于 | 单值 | 单输入框 |
| `=` | 等于 | 单值 | 单输入框 |
| `!=` | 不等于 | 单值 | 单输入框 |
| `BETWEEN` | 区间 | 双值 | 两个输入框(下限~上限) |
| `IN` | 属于列表 | 多值 | 逗号分隔输入 |

## 六、算子组件详细说明

### ① 准入/筛选条件（condition-filter）

> **分类：** 前置条件类 · **用途：** 设置谁有资格参与结算

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **名称** | `name` | 文本 | 过滤器名称，如"基准要求(时长优先豁免)" |
| **条件组** | `conditionGroup` | 条件组 | 使用通用条件组（AND/OR/NOT嵌套） |
| **未达标处置** | `failAction` | 下拉 | 见下表 |

#### 未达标处置方式（3种）

| 值 | 名称 | 说明 |
|-----|------|------|
| `eliminate` | 淘汰 | 不满足条件直接排除，不参与结算 |
| `discount` | 折算 | 按完成率打折发放奖励（展开折算配置面板） |
| `degrade` | 降级 | 降到下一档位参与结算 |

#### 折算配置（三层结构，failAction=discount 时生效）

**第一层：基础折算方式** (`discountCfg.method`)

| 值 | 名称 | 说明 | 额外配置 |
|-----|------|------|----------|
| `bucket_min` | 木桶极小值 | 取所有维度中完成率最低的值作为折算系数，如 Min(时长/60, 天数/15) | 折算分母列表（字段+目标值） |
| `ratio` | 按比例折算 | 折算系数 = 实际值 / 目标值 | 折算分母列表（字段+目标值） |
| `fixed` | 固定系数 | 不达标直接按固定比例发放，如80% | `fixedRate`（0~1之间的系数） |
| `weighted_avg` | 加权平均 | 各维度按权重加权平均作为折算系数 | 折算分母列表（字段+目标值） |

**折算分母**（bucket_min / ratio / weighted_avg 共用）

| 属性 | 类型 | 说明 |
|------|------|------|
| `denoms[].field` | 下拉 | 折算维度字段名，如"直播总时长(h)" |
| `denoms[].target` | 数字 | 该维度的目标值，如60（小时） |

**第二层：豁免规则**（可选，所有折算方式通用）

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **启用豁免** | `enableExempt` | 开关 | 开启后指定某维度达标即跳过所有折算 |
| **豁免维度** | `exemptField` | 下拉 | 豁免判断的字段名 |
| **目标值** | `exemptTarget` | 数字 | 豁免维度的目标值 |
| **豁免备注** | `exemptNote` | 文本 | 补充说明豁免规则细节 |

> 豁免逻辑：豁免维度达标 → 系数=1（全额），不走折算；豁免维度未达标 → 走第一层的基础折算方式

**第三层：附加规则**（所有折算方式通用）

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **最低折算比例** | `minRatio` | 数字(0~1) | 折算后低于此值则不予结算，如0.3=30%以下不发 |
| **最低比例说明** | `minRatioNote` | 文本 | 自定义最低比例规则描述 |
| **商业化折算说明** | `commercialNote` | 文本 | 商业化占比相关折算规则描述 |
| **折算备注** | `discountNote` | 文本 | 自由备注折算规则细节 |

### ② 计算字段定义（computed-field）

> **分类：** 前置条件类 · **用途：** 用已有字段通过公式/加权计算出新虚拟字段

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **字段名** | `fieldName` | 文本 | 新字段名称，如"影响力积分"、"综合评分" |
| **公式类型** | `formulaType` | 下拉 | `weighted_sum`(加权求和) / `custom`(自定义公式) |
| **说明** | `description` | 文本 | 该计算字段的业务含义描述 |

#### 加权求和模式（formulaType=weighted_sum）

| 属性 | 类型 | 说明 |
|------|------|------|
| `components[].field` | 下拉 | 参与加权的字段 |
| `components[].weight` | 数字 | 该字段的权重系数 |

> 公式预览：`字段名 = 字段A×权重A + 字段B×权重B + ...`

#### 自定义公式模式（formulaType=custom）

| 属性 | 类型 | 说明 |
|------|------|------|
| `customFormula` | 文本 | 自由表达式，如"付费金额 / 10"，支持引用字段名和 +、-、×、÷、() |

---

### ③ 阶梯排名发奖（rank-rule）

> **分类：** 核心规则类 · **用途：** 按指标排名分档发放奖励

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **排名方式** | `sortMethod` | 下拉 | `single`(单指标) / `weighted`(多维加权) / `formula`(公式) |
| **排序配置** | `sortFields` | 数组 | 支持多级排序，每级含字段+排序方向(desc/asc) |
| **空额累积** | `quotaOverflow` | 开关 | `cascade`(高档位未满员→剩余名额累积到低一档) / `none`(不累积) |
| **空额累积备注** | `quotaOverflowNote` | 文本 | 描述空额累积规则 |
| **档位匹配模式** | `tierMatchMode` | 下拉 | `normal`(逐档AND判断) / `min_dimension`(多维取最低值降档) |
| **金额模式** | `amountMode` | 下拉 | `fixed`(固定总额) / `unit_price`(单价×时长) |
| **时长字段** | `unitPriceField` | 下拉 | 单价模式时的时长来源字段 |
| **时长上限** | `unitPriceCap` | 数字 | 单价模式时的时长上限(小时) |
| **排名规则说明** | `rankNote` | 文本 | 特殊排名规则备注卡片 |
| **互斥排他说明** | `exclusionNote` | 文本 | 互斥排他规则备注卡片（如"本活动不与新势力叠加"） |

#### 多维加权配置（sortMethod=weighted）

| 属性 | 类型 | 说明 |
|------|------|------|
| `weightCfg[].field` | 下拉 | 参与加权的字段 |
| `weightCfg[].weight` | 数字 | 权重百分比（总和=100%） |

#### 惩罚规则（sortMethod=weighted）

| 属性 | 类型 | 说明 |
|------|------|------|
| `penalties[].field` | 下拉 | 被惩罚的字段 |
| `penalties[].action` | 文本 | 惩罚描述，如"UV奖励按70%折算" |

#### 档位表（动态增删行）

| 属性 | 类型 | 说明 |
|------|------|------|
| `tiers[].name` | 文本 | 档名，如"第1档"、"T1" |
| `tiers[].amount` | 文本 | 金额或公式（固定模式为最终金额，单价模式为小时单价） |
| `tiers[].limit` | 数字 | 该档位名额限制，留空不限 |
| `tiers[].condGroup` | 条件组 | 每档独立准入条件（复用通用条件组） |
| `tiers[].condNote` | 文本 | 档位条件备注说明 |

#### 档位匹配模式说明

| 模式 | 说明 |
|------|------|
| `normal` | 逐档AND判断，全部条件满足才入档（默认） |
| `min_dimension` | 取最低值降档——UV、时长、天数各维度分别匹配最高档位，最终取最低的作为实际档位 |

---

### ④ 阶梯达标/溢出即发（target-rule）

> **分类：** 核心规则类 · **用途：** 按指标阈值分档，达到哪档发哪档（不排名）

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **指标形态** | `metricType` | 下拉 | `absolute`(绝对值) / `overflow_ratio`(溢出比例) / `growth_rate`(增长率) |
| **基线字段** | `baseField` | 下拉 | 溢出/增长率模式时的基准字段 |
| **基线值** | `baseVal` | 数字 | 溢出/增长率模式时的基准值 |
| **规则说明** | `ruleNote` | 文本 | 特殊规则备注卡片 |

#### 指标形态说明

| 值 | 说明 | 公式 |
|-----|------|------|
| `absolute` | 直接用字段原始值判断，如ACU≥300 | 无 |
| `overflow_ratio` | 超出基线的比例 | (实际值 - 基线值) / 基线值 |
| `growth_rate` | 相对基线的增长百分比 | 同上 |

#### 档位表（动态增删行）

| 属性 | 类型 | 说明 |
|------|------|------|
| `tiers[].operator` | 下拉 | 比较运算符（支持全部8种，包括BETWEEN双值） |
| `tiers[].threshold` | 文本 | 阈值（BETWEEN时为下限） |
| `tiers[].threshold2` | 文本 | BETWEEN时的上限 |
| `tiers[].amount` | 文本 | 金额或公式 |
| `tiers[].condNote` | 文本 | 条件备注说明 |
| `tiers[].condGroup` | 条件组 | 每档独立准入条件 |

---

### ⑤ 固定金额即发（fixed-reward）

> **分类：** 核心规则类 · **用途：** 满足条件就发固定金额，不排名不瓜分

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **准入条件** | `conditionGroup` | 条件组 | 复用通用条件组（AND/OR/NOT嵌套） |
| **金额类型** | `amountType` | 下拉 | `fixed`(固定值) / `formula`(公式) |
| **金额** | `amount` | 文本 | 固定金额值 或 计算公式 |

---

### ⑥ 固定奖池平分（pool-rule）

> **分类：** 核心规则类 · **用途：** 固定一笔总奖金，由达标人分配

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **总奖池(元)** | `pool` | 数字 | 该赛道的总奖金金额 |
| **分配方式** | `dist` | 下拉 | `average`(平均) / `proportional`(按比例) / `weighted`(加权) |
| **单人上限(元)** | `cap` | 数字 | 单人最多能分到的金额上限 |

#### 分配方式说明

| 值 | 说明 |
|-----|------|
| `average` | 所有达标人均分，每人 = 总奖池 ÷ 达标人数 |
| `proportional` | 按某指标占总量的比例分配 |
| `weighted` | 多维度加权后按权值比例分配 |

---

### ⑦ 多周期奖池瓜分（multi-period-pool）

> **分类：** 核心规则类 · **用途：** 将活动期拆为多个考核周期，各周期独立奖池瓜分

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **瓜分方式** | `splitMethod` | 下拉 | `equal_share`(达标人数均分) / `weighted_share`(按指标加权分配) |
| **取整规则** | `roundRule` | 下拉 | `floor`(舍去小数) / `round`(四舍五入) / `ceil`(向上取整) |
| **瓜分说明** | `splitNote` | 文本 | 如"按照达标主播人数瓜分" |

#### 考核周期列表（动态增删）

每个周期独立配置：

| 属性 | 类型 | 说明 |
|------|------|------|
| `periods[].dateStart` | 日期 | 周期开始日期 |
| `periods[].dateEnd` | 日期 | 周期结束日期 |
| `periods[].days` | 数字 | 天数（选择日期后自动计算） |
| `periods[].conditionGroup` | 条件组 | 该周期的达标条件（复用通用条件组） |
| `periods[].pool` | 数字 | 该周期总奖池(元) |
| `periods[].cap` | 数字 | 该周期个人封顶(元) |

---

### ⑧ 流水分成/返点（share-rule）

> **分类：** 核心规则类 · **用途：** 按绑定变量的阶梯比例计算分成金额

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **绑定变量** | `variable` | 下拉 | 用于计算分成的指标字段，如"影响力积分"、"付费金额" |
| **公式(可选)** | `formula` | 文本 | 额外计算公式，如"付费金额/10" |

#### 阶梯分成表（动态增删行）

| 属性 | 类型 | 说明 |
|------|------|------|
| `tiers[].operator` | 下拉 | 比较运算符（支持BETWEEN） |
| `tiers[].threshold` | 文本 | 阈值（BETWEEN时为下限） |
| `tiers[].threshold2` | 文本 | BETWEEN时的上限 |
| `tiers[].percentage` | 数字 | 分成比例(%) |

---

### ⑨ 裂变伯乐激励（referral-rule）

> **分类：** 核心规则类 · **用途：** 老主播推荐新主播的奖励机制

| 配置项 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| **奖励发给** | `rewardTarget` | 下拉 | `referrer`(推荐人) / `referee`(被推荐人) / `both`(双方均发) |
| **触发类型** | `triggerType` | 下拉 | `headcount`(带动人数) / `revenue`(流水贡献) / `composite`(复合指标) |
| **奖励模式** | `rewardMode` | 下拉 | `per_unit`(按人/笔固定奖金) / `tiered`(阶梯表) |
| **单人/笔奖(元)** | `perUnit` | 数字 | per_unit模式时的每次奖金 |
| **个人封顶(元)** | `cap` | 数字 | 单个推荐人最多能获得的奖金上限 |
| **总预算封顶(元)** | `totalBudget` | 数字 | 所有推荐人奖金总和不超过此金额，留空不限 |
| **全局名额** | `globalQuota` | 数字 | 伯乐激励总名额限制，留空不限 |
| **超限排序字段** | `overflowSort` | 下拉 | 超名额时用此字段排名决定谁获奖 |
| **超限排序方向** | `overflowOrder` | 下拉 | `desc`(降序) / `asc`(升序) |
| **有效条件备注** | `referralNote` | 多行文本 | 描述被推荐人需满足的有效条件、作废规则等 |
| **规则说明** | `ruleNote` | 文本 | 特殊规则备注 |

#### 阶梯表（rewardMode=tiered）

| 属性 | 类型 | 说明 |
|------|------|------|
| `tiers[].amount` | 文本 | 该阶梯的奖金 |
| `tiers[].condGroup` | 条件组 | 该阶梯的准入条件 |
