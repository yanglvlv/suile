# 「另一种人生」全量重做设计文档

> Phaser 3 + 模拟经营 × 开放世界叙事
> 2026-04-10

---

## 一、核心定位

**一句话**：在一个有生命的城市里，过一段有选择的人生。

**旧版致命问题**：所有系统都是"数字表格"——点按钮→random→改数值→弹文字。没有决策深度，没有沉浸感，玩10分钟就腻。

**新版目标**：
1. **每个行动都有代价和风险**，不是简单的数字增减
2. **NPC是活的**——有自己的目标、日程、情绪、记忆
3. **城市在运转**——不因玩家而存在，玩家只是其中一员
4. **事件有因果链**——今天的决定会在未来以意外方式回响
5. **交互是游戏化的**——场景可探索、角色可移动、对话有分支

### 2.3 引擎与Phaser的边界

```
┌─────────────────────────────────────┐
│         Phaser Scene Layer          │  ← 渲染 + 输入 + 动画
│  (WorldScene, UIScene, etc.)        │
├─────────────────────────────────────┤
│         Game Engine Layer           │  ← 纯逻辑，无DOM/Canvas依赖
│  (World, Time, Character, NPCs...)  │
│  可独立单元测试                     │
├─────────────────────────────────────┤
│         Data Layer                  │  ← JSON数据，可热替换
│  (locations, npcs, events...)       │
└─────────────────────────────────────┘
```

**关键原则**：Engine层绝不引用Phaser对象。通过EventEmitter桥接：

```javascript
// Engine发出事件
world.on('player:moved', { from, to, travelTime })
world.on('npc:said', { npcId, text, emotion })
world.on('time:advanced', { oldTime, newTime })
world.on('event:triggered', { event, choices })

// Scene监听并执行渲染
this.world.on('event:triggered', ({ event, choices }) => {
    this.showEventPopup(event, choices)
})
```

## 三、核心系统重设计

### 3.1 时间系统 v2

**问题**：旧版时间只是一个计数器，除了限制地点开放时间外几乎没影响。

**重设计**：
```
时间轴: 连续流动，但按"时间块"切片处理
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│ 6-9  │ 9-12 │12-14 │14-18 │18-21 │21-24 │ 0-2  │2-6  │
│ 清晨 │ 上午 │中午 │下午 │傍晚 │夜晚 │深夜 │睡眠 │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘

关键改动:
1. 时间流逝速度可调（默认: 游戏内1小时 = 现实15秒）
2. 不同时段影响:
   - 清晨(6-9): 体力恢复快，适合锻炼/学习
   - 上午(9-12): 工作效率最高，商店营业
   - 中午(12-14): 必须吃饭，否则下午效率减半
   - 下午(14-18): 社交活跃期，NPC外出率高
   - 傍晚(18-21): 娱乐场所开放，约会黄金时间
   - 夜晚(21-24): 酒吧/夜生活，但安全度下降
   - 深夜(0-2):  部分特殊事件/NPC出现
   - 睡眠(2-6):  强制跳转，睡眠质量影响次日属性

3. 天气不只是贴图效果:
   - 雨天: 出行耗时×1.5，心情-5，但咖啡店客流+30%
   - 暴雨: 部分地点关闭，可能被困在某处触发特殊剧情
   - 雪天: 公交延误，户外活动体力消耗×2
   - 大雾: 能见度降低（UI迷雾效果），偶遇率↑

4. 季节影响:
   - 春秋: 最舒适，全属性微加成
   - 夏天: 白天更长，但高温增加疲劳
   - 冬天: 取暖费增加，部分户外活动受限，节日多
   - 年末: 总结算时刻（税务、年终奖、房租续约）
```

### 3.2 角色属性系统 v2

**问题**：旧版属性就是几个0-100的数字条，没有任何策略性。

**重设计 — 五维六性模型**：

```
┌──────── 物理层 ─────────┐
│ 💪 体力 (Energy)        │ 行动的基础消耗货币
│ ❤️ 健康 (Health)        │ 影响所有行动效率和生病风险
│ 🍔 饱腹 (Satiety)       │ 定时需求，不吃会逐级恶化      ← 新增机制
├──────── 精神层 ─────────┤
│ 😊 心情 (Mood)          │ 影响社交结果和事件触发倾向
│ 🧠 压力 (Stress)        │ 累积型，高压力触发负面连锁
│ ✨ 动机 (Motivation)     │ 替代"意志力"，影响行动成功率  ← 重命名重构
├──────── 社会层 ─────────┤
│ ⭐ 声望 (Reputation)     │ 解锁新地点/新NPC/新机会
│ 💳 信用 (Credit)        │ 影响贷款/租房/求职
│ 🔗 人脉 (Connections)   │ 具体的关系网络图谱，非单一数值│ ← 从NPC系统衍生
└──────── 经济层 ─────────┘
│ 💵 现金 (Cash)           │ 即时消费
│ 🏦 存款 (Savings)       │ 有利息，取款需T+1
│ 📊 总资产 (Net Worth)    │ 成就/结局判定依据
└─────────────────────────┘
```

**核心机制改变**：

#### (1) 饱腹系统 — 三餐机制
```
不是简单的 hunger++，而是:
- 早餐(6-10): 不吃 → 上午效率-20%，心情-5
- 午餐(11-14): 不吃 → 下午效率-40%，健康-2
- 晚餐(17-21): 不吃 → 夜间无法恢复体力，失眠概率+30%
- 吃什么有区别:
  · 便利店便当($15): 饱腹+40, 微量健康-1
  · 家常菜(自己做): 饱腹+50, 心情+5, 技能exp+
  · 餐厅吃饭($80): 饱腹+60, 心情+10, 可能遇NPC
  · 不吃/忘记: 进入饥饿状态，3级衰减
```

#### (2) 动机系统 — 替代死板的意志力
```
动机来源(叠加):
· 睡眠充足(+20~40) · 完成目标奖励(+15) · 社交满足(+10)
· 有趣的活动(+10) · 金钱压力(-15) · 连续加班(-25)

动机影响:
· 高动机(>70): 行动成功时有额外收益(经验×1.5, 偶发灵感)
· 正常(30-70): 标准表现
· 低动机(<30): 行动可能失败! (如: 锻炼→中途放弃, 学习→学不进去)
· 极低(<10): 角色拒绝执行非必要行动，需要先休息/娱乐
```

#### (3) 状态异常系统
```
不仅仅是 sick=true/false:

🤒 轻感冒: 效率-20%, 传染给同居室友概率
🤢 肠胃炎: 只能在家/医院, 2天不能出门
😵 过劳晕倒: 在任意行动中随机触发, 被送回家强制休息
😰 焦虑发作: 社交行动失败率+50%, 选择受限
🍺 宿醉: 头痛Debuff持续到当天结束, 昨晚喝多了的后果
💔 心碎: 分手/被拒后, 社交动力暴跌, 持续数天
```

### 3.3 场景/地图系统 v2

**问题**：旧版就是一张背景图+一个列表，没有任何探索感。

**重设计 — 多层次场景**：

```
城市视图(大地图):
┌─────────────────────────────────────┐
│  ┌─────┐  ┌─────────┐  ┌───────┐  │
│  │老城区│  │   CBD   │  │市中心 │  │
│  │ 🏚️  │  │  🏢🏢   │  │ 🛒🎬  │  │
│  └──┬──┘  └────┬────┘  └───┬───┘  │
│     │  ┌──────────┐     │        │
│     └─→│ 文创区   │←────┘        │
│        │  ☕🎨     │              │
│        └──────────┘              │
└─────────────────────────────────────┘

点击区域进入 → 场景内部(Tiled地图):

示例: 街角咖啡店场景 (cafe.tmx)
┌──────────────────────────┐
│ ░░░░░░░░░░░░░░░░░░░░░░░ │  ░=墙/不可通行区域
│ ░  🚪              🖼️ ░ │  🚪=出口(回到大地图)
│ ░ ┌────┐  ☕☕   ┌────┐ ░ │  ☕=吧台交互点
│ ░ │桌1 │        │桌2 │ ░ │  🪑桌子=坐下互动
│ ░ └────┘  👨‍🍳   └────┘ ░ │  👨‍🍳=林夕(NPC)
│ ░    🪑    📚        🪑  ░ │  📚=书架交互
│ ░░░░░░░░░🚪░░░░░░░░░░░░░░░ │
└──────────────────────────┘

玩家可以用虚拟摇杆/点击在场景内走动
走到NPC身边 → 弹出互动选项
走到交互物旁边 → 显示可用行动
靠近门口 → 提示离开
```

**关键设计原则**：
1. **每个场景都不止一个用途** —— 咖啡店可以: 喝咖啡/工作/看书/遇到NPC/听八卦/打工
2. **空间有意义** —— 坐靠窗的位置可能看到街景事件，坐吧台可以和咖啡师聊天
3. **不同时间去不一样** —— 早上安静只有常客，中午爆满需要等位，晚上有驻唱

### 3.4 NPC系统 v2 — 最大重写

**问题**：旧版NPC就是随机句子生成器。

**重设计 — 有灵魂的AI**：

#### (1) NPC不再是对话框里的头像，是场景里走来走去的人
```
每个NPC在场景中有:
· 独立精灵（外观随衣着/心情微变）
· 巡逻路径/停留点（不是站桩）
· 自主行为AI（不是等玩家来点）

例如林夕(咖啡师):
08:00-10:00 → 站在吧台后面擦杯子（工作状态）
10:00-10:15 → 坐角落休息看手机（可搭话）
14:00-16:00 → 在店里走动整理（忙碌，不太想聊）
20:00之后 → 下班，可能在酒吧或者已经回家
```

#### (2) 对话树系统 — 替换随机句子
```json
{
  "id": "linxi_first_meet",
  "root": {
    "text": "（她正在擦拭一只精致的咖啡杯，注意到你走近）",

  },
  "nodes": {
    "greeting": {

**对话树 vs 旧版对比**：
| | 旧版 | 新版 |
|--|------|------|
| 对话内容 | 5句模板随机轮换 | 树状结构，有分支有条件 |
| 玩家选择 | 6个固定选项 | 根据情境/属性/关系动态变化 |
| 记忆感 | 没有 | 第一次见面vs第十次见面完全不同 |
| 后果 | 只是数值+2 | 可能解锁新信息/新任务/新路线 |

#### (3) NPC有自己的生活
```
NPC每日循环:
起床 → 洗漱 → 通勤 → 工作 → 午休 → 工作 → 下班 → 
自由时间(购物/社交/娱乐/回家) → 睡眠

关键: NPC的行为会影响世界!
· 小王今天请了病假 → 公司项目延期 → 张哥心情不好来找你麻烦
· 老陈的面摊今天卫生检查没过 → 关停3天 → 你只能去别处吃饭
· 美美发了一条朋友圈@了你 → 其他NPC看到了会有反应
```

#### (4) 关系系统的真实演进
```
陌生人 → 面熟 → 认识 → 朋友 → 密友 → (可选: 暧昧/恋人/仇敌)

每级解锁:
面熟: 知道名字, 打招呼有回应
认识:  知道职业/住址, 可以约出来
朋友:  知道手机号, 会主动联系, 可以求助
密友:  知道秘密, 会在困难时帮你, 但也会对你失望

关系可以退化!
· 长时间不联系 → 降级
· 做出违背对方价值观的事 → 大幅降级甚至变敌对
· 被发现说对方坏话(口碑传播) → 关系破裂
```

### 3.5 事件系统 v2

**问题**：22个事件全部 `Math.random() < 阈值`，没有叙事感。

**重设计 — 叙事驱动的事件引擎**：

#### (1) 事件分类重新定义
```
A. 日常小事 (频率高, 影响小)
   · 路上踩到水坑 / 公交卡余额不足 / 发现一家新店
   · 作用: 让世界 feel alive，打破单调
   
B. 个人危机 (有前置条件, 链式触发)
   · 加班过度 → 身体不适 → 请假 → 绩效下降 → 被约谈
   · 投资亏损 → 焦虑 → 和朋友喝酒 → 说错话 → 关系受损
   · 作用: 让玩家感受到选择的重量
   
C. 人生转折 (低频率, 高影响, 有分支选择)
   · 收到猎头电话 / 房东要卖房 / NPC表白 / 家人突发状况
   · 作用: 改变游戏走向的关键节点
   
D. 世界事件 (与玩家无关, 但玩家可以选择介入或不介入)
   · 城市举办美食节 / 某公司裁员潮 / 网红打卡地开业
   · 作用: 世界在独立运转的感觉
   
E. 彩蛋/奇遇 (极低概率, 独特体验)
   · 买彩票中了(真的大奖) / 在公园捡到一只狗 / 
     遇到一个长得和你一模一样的人
```

#### (2) 因果链系统 — 真正的链条
```
示例链1: "过度工作的代价"
  [连续工作7天以上] 
    → [事件:同事说你最近脸色很差] (警告)
    → [无视警告继续加班]
    → [事件:在工位上差点睡着,被张哥看到] (声誉-5)
    → [仍然不休息]
    → [事件:晕倒在办公室,被救护车拉走] (强制住院3天, 医疗费$2000)
    → [出院后发现岗位被人顶替了] (可能失业!)

每一步都有岔路:
  [同事说你脸色差]
  → A. "没事,我能扛" → 继续链
  → B. "你说得对,今天早走" → 断链, 恢复正常
  → C. "帮我跟老板说说" → 触发另一条支线(同事关系线)

链的状态存储在 chainState 中，跨天持久化。
```

#### (3) 事件的呈现方式
```
不再是弹出一行文字消失!

小事件 → 浮动通知(现有showFloatMsg保留优化)
中等事件 → 全屏卡片 + 选择分支
重大事件 → 过场动画 + 多段对话 + 延迟后果
世界事件 → HUD新闻滚动条 + 地图标记 + 可选参与
转折事件 → 专门的"章节"界面, 类似视觉小说
```

### 3.6 经济/职业/技能系统 v2

**经济系统**：
```
收入来源:
· 工资(月薪制,每月1号发放) — 主业
· 副业(兼职/外包/摆摊) — 需要投入时间
· 投资(股票/基金/房产) — 有风险
· 偶然收入(卖东西/中奖/红包) — 不可控

支出项:
· 固定支出: 房租(每月), 交通费(通勤), 通讯费
· 浮动支出: 餐饮(看你吃什么), 娱乐, 购物
· 突发支出: 生病/修车/送礼/罚款
· 大额支出: 买房/买车/进修/旅游

关键改动:
· 股票不再是纯随机 — 受游戏内新闻事件影响(某公司裁员→股价跌)
· 可以负债! — 信用卡/网贷/向NPC借钱, 还不上有后果
· 有账单日 — 每月固定日期扣款,钱不够就逾期
```

**职业系统**：
```
不是写死的"程序员$4500":

求职流程:
浏览招聘APP → 筛选(匹配技能/学历要求) → 投简历 → 
等待(1-3天随机) → 面试(小游戏:回答问题/压力测试) → 
录用/拒绝 → 入职培训 → 试用期(绩效考核) → 转正/辞退

工作中:
每天有任务(完成KPI/处理突发事件/应付难缠客户)
工作态度选择影响: 绩效/同事关系/上司看法/自身压力
可能发生: 加薪/升职/调岗/裁员/挖槽/创业机会

副业选择:
· 外卖骑手(时间灵活,收入不稳定)
· 自媒体(前期0收入,爆发性强)
· 家教(时薪高,需要学历/耐心)
· 摆摊(低成本,城管风险哈哈)
· 直播(需要魅力/口才/运气)
```

**技能系统 — 有实际用处**：
```
每种技能影响具体能力:

编程Lv1: 能看懂代码 | Lv3: 能接简单外包 | Lv5: 能做出产品 | Lv8: 创业选项解锁
烹饪Lv1: 做饭不难吃 | Lv3: 做饭省钱又好吃 | Lv5: 可以开小吃摊 | Lv8: 可以开餐厅
健身Lv1: 不容易累 | Lv3: 体格优势(某些事件) | Lv5: 可以当教练 | Lv8: 可以打比赛
社交Lv1: 不怯场 | Lv3: 快速提升好感 | Lv5: 获得隐藏信息 | Lv8: 人脉即金钱
驾驶Lv1: 能开车 | Lv3: 开车省时间 | Lv5: 可以跑滴滴 | Lv8: 可以赛车(?)

技能组合解锁:
编程+商业 = 创业(做自己的产品)
烹饪+社交 = 开店
健身+社交 = 当教练/网红
写作+社交 = 自媒体大V
谈判+商业 = 中介/销售精英
```

## 四、UI/UX 重设计

### 4.1 视觉风格
```
风格方向: 2D俯视角 + 像素风/精致像素 (不是粗糙像素!)
