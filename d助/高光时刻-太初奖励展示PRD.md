# 高光时刻 - 新增太初奖励展示 PRD

## 一、需求背景

当前D助「高光时刻」模块仅支持**增幅**、**强化**和**土罐**三类高光记录的展示与分享。太初装备作为游戏中重要的装备获取途径，获得太初装备同样是玩家值得记录和炫耀的高光时刻，但目前缺少对应的展示入口。

本次需求旨在：
1. 高光时刻详情页新增「太初」Tab，支持太初装备获取记录的展示与分享
2. D助首页「高光时刻」模块入口增加**红点提示**，当有新的高光数据（增幅/强化/土罐/太初）时提醒用户查看

---

|------|------|

## 三、功能详细设计

### 3.1 首页入口 - 红点提示（新增）

**位置**：D助首页「高光时刻」标题右侧

**交互规则**：

| 规则 | 说明 |
|------|------|
| 红点出现条件 | 用户有**未查看**的新高光时刻记录（包含增幅/强化/太初任一类型） |
| 红点消除条件 | 用户**点击进入**高光时刻详情页后，红点消失 |
| 红点样式 | 标题右侧小红点（无数字），参考通用红点规范 |
| 刷新机制 | 每次进入D助首页时请求接口判断是否有新数据 |

**示意**：

```
┌──────────────────────────────────────┐
│  高光时刻 🔴                     >>  │
│                                      │
│          暂无高光时刻                  │
│            🐼                        │
│                                      │
│  团本记录                             │
└──────────────────────────────────────┘
```

> 🔴 = 红点，表示有新的高光时刻数据未查看

---

### 3.2 高光时刻详情页 - 新增太初Tab

**Tab栏变更**：

| 变更前 | 变更后 |
|--------|--------|
| 全部 \| 增幅 \| 强化 \| 土罐 | 全部 \| **太初** \| 增幅 \| 强化 \| 土罐 |

**Tab交互**：
- 新增「太初」Tab，点击筛选仅展示太初类型的高光时刻
- 「全部」Tab 需包含太初类型记录，按时间倒序混排（增幅/强化/土罐/太初）
- Tab样式与现有增幅、强化、土罐一致

---

### 3.3 太初高光时刻卡片

复用现有「闪光时刻」卡片模板，调整展示元素：

**卡片内容对比**：

| 元素 | 增幅/强化（现有） | 太初（新增） |
|------|------------------|-------------|
| 标题 | · 闪光时刻 · | · 闪光时刻 · |
| 装备图标 | 增幅/强化装备icon | **太初装备icon** |
| 装备名称 | 装备名称 | **太初装备名称** |
| 操作描述 | 增幅大成功！/ 强化大成功！ | **太初获取！** |
| 等级信息 | 增幅+12 / 强化+13 | **无（或展示装备等级）** |
| 角色信息 | 区服 \| 角色名 \| 转职名(性别) | 同左 |
| 时间 | 日期 | 同左 |
| 二维码 | 有 | 同左 |

**卡片示意**：

```
┌─────────────────────────┐
│     · 闪光时刻 ·          │
│                           │
│    [太初装备icon]          │
│    太初装备名称            │
│    太初获取！              │
│                           │
│    角色名                  │
│    区服 | 转职名(性别)      │
│    2026.01.10              │
│                       [QR] │
└─────────────────────────┘
```

## 四、接口需求

### 4.1 红点状态接口

| 项目 | 说明 |
|------|------|
| 接口用途 | 查询用户是否有未查看的新高光时刻 |
| 调用时机 | 进入D助首页时 |
| 入参 | 用户ID、角色ID |
| 出参 | has_new（bool）：是否有新数据 |

### 4.2 红点已读上报接口

| 项目 | 说明 |
|------|------|
| 接口用途 | 用户进入高光时刻详情页后，上报已读状态 |
| 调用时机 | 进入高光时刻详情页时 |
| 入参 | 用户ID、角色ID |
| 出参 | 成功/失败 |

### 4.3 高光时刻列表接口（变更）

| 项目 | 说明 |
|------|------|
| 变更内容 | type字段新增枚举值「太初」 |
| 筛选支持 | 全部 / 增幅 / 强化 / 土罐 / **太初** |
| 太初数据字段 | 装备icon_url、装备名称、获取时间、角色信息 |

## 五、埋点需求

| 埋点事件 | 说明 |
|----------|------|
| highlight_redpoint_show | 红点展示（首页） |
| highlight_redpoint_click | 红点点击进入详情页 |
| highlight_tab_click | Tab切换，参数：tab_name（全部/增幅/强化/土罐/太初） |
| highlight_taichu_card_view | 太初卡片曝光 |
| highlight_taichu_share | 太初卡片分享，参数：share_channel |

---

## 六、异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 太初Tab无数据 | 展示空态：「暂无高光时刻」+ 熊猫图标（同现有） |
| 红点接口超时 | 不展示红点，不影响主流程 |
| 装备icon加载失败 | 展示默认占位图 |
| 多角色切换 | 红点状态跟随当前角色，切换角色后重新查询 |

---

## 七、改动范围总结

```
改动点：
├── D助首页
│   └── 高光时刻入口 → 新增红点提示逻辑
├── 高光时刻详情页
│   ├── Tab栏 → 新增「太初」Tab（位于全部之后）
│   ├── 列表 → 全部Tab混排太初数据
│   └── 卡片 → 新增太初类型卡片（复用模板，替换icon+名称）
└── 分享图
    └── 太初类型分享图（复用闪光时刻模板）
```
