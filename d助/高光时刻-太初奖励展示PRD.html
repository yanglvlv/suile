一、需求背景

当前D助「高光时刻」模块仅支持增幅、强化和土罐三类高光记录的展示与分享。太初装备作为游戏中重要的装备获取途径，获得太初装备同样是玩家值得记录和炫耀的高光时刻，但目前缺少对应的展示入口。

本次需求目标：

  1. 高光时刻详情页新增「太初」Tab，支持太初装备获取记录的展示与分享
  2. D助首页「高光时刻」模块入口增加红点提示，当有新的高光数据（增幅/强化/土罐/太初）时提醒用户查看

二、功能详细设计

2.1 首页入口 - 红点提示 [新增]

位置：D助首页「高光时刻」标题右侧

交互规则

规则说明
红点出现条件用户有未查看的新高光时刻记录（包含增幅/强化/土罐/太初任一类型）
红点消除条件用户点击进入高光时刻详情页后，红点消失
红点样式标题右侧小红点（无数字），参考通用红点规范
刷新机制每次进入D助首页时请求接口判断是否有新数据

红点流程

进入D助首页 → 请求红点接口 → has_new=true → 展示红点 → 用户点击进入详情页 → 上报已读 → 红点消失

2.2 高光时刻详情页 - 新增太初Tab [新增]

Tab栏变更

变更前变更后
全部 | 增幅 | 强化 | 土罐全部 | 太初 | 增幅 | 强化 | 土罐

Tab交互规则

Tab行为
全部展示所有类型，按时间倒序混排（太初/增幅/强化/土罐）
太初仅展示太初装备获取记录（新增），样式与其他Tab一致
增幅仅展示增幅类型记录（同现有）
强化仅展示强化类型记录（同现有）
土罐仅展示土罐类型记录（同现有）

三、埋点需求 [新增]

说明：高光时刻模块此前无埋点，本次需整体补全埋点方案，覆盖全部Tab类型（太初/增幅/强化/土罐）。

3.1 页面访问（UV / PV）

埋点事件触发时机关键参数统计指标
highlight_page_view用户进入高光时刻详情页-页面 PV / UV

3.2 Tab切换

埋点事件触发时机关键参数统计指标
highlight_tab_click用户点击/切换Tabtab_name：全部 / 太初 / 增幅 / 强化 / 土罐各Tab切换人数（UV）、次数（PV）

3.3 分享

埋点事件触发时机关键参数统计指标
highlight_share_click用户点击分享按钮card_type：太初 / 增幅 / 强化 / 土罐
share_channel：保存本地 / 动态 / 关注 / 微信 / 朋友圈 / QQ空间		分享次数（按类型、渠道拆分）

3.4 埋点参数汇总

参数名类型说明枚举值
tab_namestring当前所在Tab全部 / 太初 / 增幅 / 强化 / 土罐
card_typestring卡片类型（高光类型）太初 / 增幅 / 强化 / 土罐
share_channelstring分享渠道保存本地 / 动态 / 关注 / 微信 / 朋友圈 / QQ空间

3.5 数据看板需求

基于以上埋点，需要支持以下数据查看维度：

序号看板指标说明
1高光时刻页面 UV / PV整体访问量趋势
2各Tab切换 UV / PV按 tab_name 拆分，了解各类型受关注程度
3分享次数 - 按类型按 card_type 拆分，了解哪类高光时刻分享意愿最高
4分享次数 - 按渠道按 share_channel 拆分，了解用户偏好的分享渠道
5分享次数 - 类型×渠道交叉交叉分析各类型在各渠道的分享分布

四、改动范围总结
