一、需求背景

当前资讯库支持两种方式导出资讯数据（资讯库导出 / 按时间维度导出），导出的 Excel 中包含新标题、资讯id、缩略图、原缩略图、新缩略图、作件/状态、原资讯链接、新资讯链接、发布时间、国标时间、类型、平台、作者、来源、浏览量、评论量、频道名称、资讯位置、资讯来源、资讯类型等字段。

现有导出数据中缺少「关键词」字段。关键词在「游戏说审核」页面中已有配置入口，每条资讯可配置一个或多个关键词（如"版本前瞻"、"活动攻略"等）。运营需要在导出数据时能够看到每条资讯对应的关键词信息，便于后续的数据分析和内容归类。

本次需求目标：

  1. 在资讯导出的 Excel 中新增多列「关键词」字段（关键词一、关键词二、关键词三……），每个关键词独占一列
  2. 关键词数据来源于游戏说审核中配置的关键词字段
  3. 多列展示便于导出后直接按列筛选、排序、归类统计，无需二次拆分

二、现状说明

2.1 资讯导出方式

目前有两种导出方式，导出的字段结构基本一致，仅时间维度筛选不同：

导出方式说明时间维度
方式一：资讯库导出从「资讯管理 → 资讯库」页面导出，可覆盖长时间范围的资讯内容导出（超过1年）支持自定义起止时间，范围较大
方式二：按时间维度导出同样从资讯管理导出，字段结构与方式一一致按较短时间维度筛选导出

2.2 关键词数据来源

关键词来源于「游戏说审核」页面：

  • 每条资讯在游戏说审核页面的右侧面板中有「关键词」配置区域
  • 支持添加多个关键词（通过「+ 添加关键词」按钮逐个添加）
  • 每个关键词支持单独删除（关键词旁有 × 按钮）
  • 提供「清空」按钮可一次性清除所有关键词
示例：一条资讯可能配置了关键词 "版本前瞻"、"活动攻略"、"新职业" 三个关键词。

三、功能详细设计

3.1 导出字段变更 [新增]

新增字段 — 多列展示

采用多列展示方案，每个关键词独占一列，便于导出后直接在 Excel 中按列筛选、排序和归类统计。

字段名列位置数据来源说明
关键词一现有末尾列 +1读取游戏说审核中该资讯配置的关键词数据，按配置顺序依次填入各列第1个关键词
关键词二现有末尾列 +2第2个关键词（如无则留空）
关键词三现有末尾列 +3第3个关键词（如无则留空）
…依此类推列数根据所有资讯中关键词的最大数量动态确定

列数确定规则

规则说明
动态列数导出时，先遍历所有待导出资讯，取关键词数量的最大值 N，生成 N 列（关键词一 ~ 关键词N）
不足补空某条资讯关键词数量不足 N 个时，多余的列留空
无关键词若该资讯无关键词，所有关键词列均留空
填充顺序按游戏说审核中关键词的配置顺序，从「关键词一」开始依次填充

各场景显示效果

场景关键词一关键词二关键词三说明
无关键词（空）（空）（空）所有关键词列留空
1个关键词版本前瞻（空）（空）仅关键词一有值
2个关键词版本前瞻活动攻略（空）关键词一、二有值
3个关键词版本前瞻活动攻略新职业依此类推

3.2 导出 Excel 示例 [变更]

以下为导出 Excel 的列结构变更示意（仅展示末尾新增列部分）：

… 资讯来源 资讯类型 关键词一 [新增] 关键词二 [新增] 关键词三 [新增]
… SC_INFO info 版本前瞻
… SC_INFO info 版本前瞻 活动攻略
… SC_INFO info 版本前瞻 活动攻略 新职业
… VIDE info （空） （空） （空）
… SC_INFO info moment
说明：上例中最大关键词数量为 3，因此生成「关键词一」「关键词二」「关键词三」共 3 列。实际列数根据导出数据动态确定。

3.3 两种导出方式均需支持 [变更]

两种资讯导出方式（资讯库导出 / 按时间维度导出）均需要新增关键词多列，保持导出字段一致性：

导出方式是否新增关键词列备注
方式一：资讯库导出是 ✅长时间范围导出，同样需要关键词多列字段
方式二：按时间维度导出是 ✅短时间维度导出，同样需要关键词多列字段

四、数据获取逻辑

4.1 关键词数据读取

属性说明
数据来源游戏说审核系统中，每条资讯对应的关键词配置
关联方式通过资讯id关联，获取该资讯在游戏说审核中配置的关键词列表
数据格式关键词为数组形式，如 ["版本前瞻", "活动攻略", "新职业"]
填充规则数组第1个元素 → 关键词一列，第2个 → 关键词二列，依此类推
空值处理如果该资讯无关键词配置，所有关键词列留空

4.2 处理流程

导出触发
  → 按筛选条件查询资讯列表
  → 遍历每条资讯，通过资讯id查询游戏说审核中的关键词数据
  → 统计所有资讯中关键词数量的最大值 N
  → 生成表头：关键词一、关键词二、…、关键词N
  → 遍历每条资讯，将关键词数组按顺序填入对应列（不足 N 个的列留空）
  → 导出文件

五、影响范围

5.1 改动范围

改动点：
├── 管理端 - 资讯导出功能
│   ├── 方式一（资讯库导出） → 新增「关键词一～关键词N」多列
│   └── 方式二（按时间维度导出） → 新增「关键词一～关键词N」多列
└── 后端接口
    └── 导出接口 → 增加查询游戏说审核关键词数据的逻辑，动态生成多列输出到 Excel

5.2 注意事项

  • 性能：导出数据量较大时（如超过1年的数据），关键词查询需要注意查询效率，建议批量查询而非逐条请求
  • 兼容性：历史资讯如果在游戏说审核中没有配置关键词，导出时关键词列留空即可，不影响其他字段
  • 一致性：两种导出方式的关键词列数量、位置和格式需保持一致
  • 动态列数：关键词列数根据本次导出数据中关键词的最大数量动态确定，不同批次导出的列数可能不同
