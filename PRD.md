## 1. 产品概述

### 1.1 产品定位

DNF 官方壁纸站是地下城与勇士（DNF）游戏官方的**高清壁纸展示与分发平台**，面向全平台用户提供：

- **官方壁纸**：版本KV主视觉、人物立绘、活动/节日限定壁纸等多分辨率下载
- **同人画廊**：社区创作者投稿的优质同人作品展示

### 1.2 目标用户

| 用户角色 | 说明 | 核心需求 |
|---|---|---|
| **C端玩家** | DNF 游戏玩家 / 壁纸爱好者 | 浏览、预览、下载多分辨率壁纸 |
| **运营人员** | 官网内容运营 | 管理Banner、分类、壁纸内容、配置投稿规则 |

### 1.3 参考产品

- 米游社同人图发布/展示系统
- 原神官方壁纸站（布局、分类结构、下载面板）
- DNF 各版本KV视觉素材

## 3. 功能详细说明

### 3.1 首屏 / 导航区

#### 3.1.1 固定顶部导航栏

| 属性 | 说明 |
|---|---|
| **位置** | 固定页面顶部，毛玻璃效果 (`backdrop-filter: blur(20px)`)，高度 60px |
| **左侧** | Logo：**DNF 壁纸站 / WALLPAPER GALLERY** |
| **中间** | 一级分类导航（动态渲染） |
| **右侧** | 全局搜索框 + 搜索图标 |

#### 3.1.2 一级分类导航

支持两种类型的一级分类，由后台配置：

| 类型 | 名称 | 行为差异 |
|---|---|---|
| `normal` | 官方壁纸 | 展示二级分类Tab；不显示投稿入口 |
| `ugc` | 同人画廊 | 不展示二级分类Tab；显示投稿入口横幅 |

**交互行为：**
- 点击切换一级分类 → 刷新下方二级Tab和壁纸列表，页码重置为1
- 当前项高亮（蓝色文字 + 底部下划线指示器）
- Hover 时背景微变

**默认分类结构：**
```
官方壁纸 (normal, id=1)
  ├── 版本KV (id=11)
  ├── 人物立绘 (id=12)
  └── 活动/节日壁纸 (id=13)
同人画廊 (ugc, id=2)
```

#### 3.1.3 全局快捷搜索

- 触发方式：输入后按 Enter 键
- 搜索范围：**跨所有分类**搜索壁纸标题 + 发布日期 + 扩展字段值（如版本名称、创作者昵称）
- 清空搜索框后按 Enter 恢复当前分类列表

---

### 3.2 Hero Banner 轮播区

| 属性 | 说明 |
|---|---|
| **尺寸** | 全宽 × 340px 高度，圆角底部 20px |
| **播放机制** | 自动轮播，间隔 5 秒，淡入淡出过渡 (1s ease) |
| **每张Banner包含** | 背景大图、标签(tag)、标题、描述文字 |
| **跳转** | 支持配置外部链接（可选） |
| **控制组件** | 底部进度条指示器 + 左右箭头按钮 |
| **渐变遮罩** | 顶部和底部各一层半透明渐变，保证文字可读性 |
| **数量** | 不限数量，后台自由配置 |

**数据字段：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | Number | Y | 唯一标识 |
| img | String | Y | Banner 图片URL (1400×520) |
| title | String | Y | 标题文字 |
| desc | String | N | 描述副标题 |
| tag | String | N | 左下角标签（如"最新版本"） |
| link | String | N | 点击跳转链接 |
| sort | Number | Y | 排序权重（升序） |
| status | Boolean | Y | 是否启用 |

---

### 3.3 二级分类 Tab 栏

| 属性 | 说明 |
|---|---|
| **位置** | Banner 下方，最大宽度 1320px 居中 |
| **样式** | 圆角胶囊形 Tab，未选中灰色边框透明底，选中蓝底白字 |
| **默认项** | "全部"按钮（显示当前一级分类下所有子分类的壁纸） |
| **动态性** | 根据选中的一级分类动态渲染其 children 子分类列表 |
| **无子分类时** | 不渲染 Tab 栏（如同人画廊无子分类，则直接展示壁纸） |

### 3.4 壁纸卡片网格

#### 3.4.1 布局规格

| 属性 | 值 |
|---|---|
| 列数 | 桌面端 3 列，平板端 2 列 |
| 卡片宽高比 | 16:9 (padding-top: 56.25%) |
| 间距 | 20px |
| 圆角 | 14px |
| 边框 | 1px solid #eaeef4 |
| 分页 | 每页 9 条，底部分页器 |

#### 3.4.2 卡片结构

```
┌─────────────────────────────┐
│  [角标]                      │  ← 可选，左上角 NEW/HOT/精选
│                             │
│     缩略图 (16:9)           │
│                             │
│   [🔍] [⬇]                 │  ← hover 显示操作按钮
│  ─────────────────          │
│  壁纸标题                    │
│  日期 · 元数据               │  ← 版本名称 / 角色 / 创作者等
│  👤 作者头像 昵称             │  ← 仅 UGC 分类且有 authorAvatar 时显示
└─────────────────────────────┘
```

#### 3.4.3 Hover 交互

- 整卡上浮 4px + 加深阴影
- 缩略图微放大 4% (scale 1.04)
- 显示半透明遮罩层（从上到下 30%→70% 渐变）
- 右上角浮现「预览」「下载」操作按钮
- 底部显示标题+元数据覆盖层

#### 3.4.4 入场动画

- `fadeUp` 关键帧动画：从下方 16px 透明度0滑入
- 每张卡片依次延迟 `i * 0.04s`

#### 3.4.5 同人画廊特殊展示

当壁纸属于 UGC 分类且配置了以下字段时，卡片额外显示：
- **authorAvatar**: 圆形头像 (20×20px)，使用 pravatar 或自定义 URL
- **创作者扩展字段**: 在 extFields 中 key 为"创作者"的字段值作为昵称

### 3.5 投稿入口横幅

| 条件 | 说明 |
|---|---|
| **显示条件** | 仅当前一级分类 type 为 `ugc` 且 submitConfig.status 为 true 时显示 |
| **样式** | 渐变背景卡片（蓝紫渐变），左侧文案 + 右侧 CTA 按钮 |
| **可配置项** | 横幅标题、描述文字、按钮文案（均来自 submitConfig） |
| **点击行为** | 弹出投稿规则弹窗（非实际提交功能，仅展示规则+邮箱引导） |

**投稿弹窗内容：**
- 标题 + 描述
- 投稿要求（格式、最低分辨率）
- 投稿规则（多条，换行展示）
- 引导至邮件投稿

#### 3.6.2 预览区域功能

| 功能 | 说明 |
|---|---|
| **默认图片** | 始终先加载缩略图放大版（节省带宽） |
| **显示原图按钮** | 右上角，仅当壁纸有 `originalImg` 字段时可见；点击后加载原图并隐藏按钮 |
| **左右切换** | 箭头按钮 + 键盘方向键 ← → |

#### 3.6.3 侧栏信息面板

| 区域 | 内容 |
|---|---|
| **标题** | 壁纸名称（18px 粗体） |
| **日期** | 发布日期（12px 灰色） |
| **分类归属** | 一级分类名 / 二级分类名 |
| **扩展字段** | 动态展示所有 `show=true` 的 extFields（key-value 对） |

#### 3.6.4 下载列表

- 从壁纸数据的 `resFiles[]` 数组读取
- 按 `type` 字段分为两组：「横版尺寸」「竖版尺寸」
- 每项显示分辨率 label + 下载图标按钮
- 有 URL 则 `window.open()` 新窗口打开；无则 alert 提示
- 无任何 resFiles 时提示"暂无可下载的分辨率"

| 能力 | 说明 |

#### 4.2.3 壁纸管理

**列表字段：**
缩略图、标题、分类归属、扩展信息、角标(Badge)、状态、操作

**编辑弹窗表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 标题 | text | Y | 壁纸名称 |
| 分类 | select | Y | 下拉选择分类（一级或二级） |
| 缩略图URL | text | Y | 卡片展示用图 |
| 原图URL（高清大图） | text | N | 弹窗预览时可切换查看；留空则无"显示原图"按钮 |
| 作者头像 | text | N | 同人画廊专用，圆形头像URL |
| 发布时间 | text | Y | 格式 YYYY/MM/DD |
| 角标 | text | N | 如 NEW / HOT / 精选 |
| 排序 | number | Y | 升序排列权重 |
| 启用状态 | select | Y | 是否在前台展示 |
| **扩展字段** | 动态键值对 | N | 可自由添加 key/value，勾选是否在前台展示 |
| **多分辨率图片** | 动态行 | N | 每行包含：分辨率标签、横/竖版选择、图片URL |

**分辨率编辑说明：**
- 弹窗底部提供**预设快捷按钮**（1920×1080、2560×1440、3840×2160、1080×1920 等 8 个常用尺寸）
- 点击预设按钮自动添加一行（已添加的预设会从按钮列表中隐藏）
- 也可点击「+ 添加分辨率」手动输入自定义尺寸
- 每行可单独删除
- 保存时从弹窗 DOM 实时收集所有分辨率行数据

**保存逻辑：**
- 新增时生成自增 ID（壁纸 ID 从 101 起步，避免与分类 ID 冲突）
- 编辑时完整覆盖 resFiles（从弹窗 DOM 实时收集，不丢失数据）
- originalImg 和 authorAvatar 作为独立字段保存

#### 4.2.4 投稿配置

| 图片格式 | format | 要求的文件格式 |

## 5. 数据模型

### 5.1 存储方案

- **存储介质**：浏览器 `localStorage`
- **统一 Key**：`dnf_wallpaper_config`
- **前后台同步**：admin.html 写入 → index.html 读取同一 Key
- **数据初始化**：首次访问无 localStorage 数据时，使用 `getDefaultData()` 内联默认值

### 5.2 数据结构定义

```javascript
{
  // ===== Banner 数组 =====
  banners: [
    {
      id: Number,           // 唯一标识（自增）
      img: String,          // 图片URL (1400×520)
      title: String,        // 标题
      desc: String,         // 描述副标题
      tag: String,          // 标签文字（如"最新版本"）
      link: String,         // 跳转链接（可选）
      sort: Number,         // 排序权重
      status: Boolean,      // 是否启用
    }
  ],

  // ===== 分类数组（树形结构）=====
  categories: [
    {
      id: Number,           // 一级分类ID（1, 2, ...）
      name: String,         // 分类名称
      type: 'normal' | 'ugc', // 分类类型
      sort: Number,         // 排序权重
      status: Boolean,      // 是否启用
      children: [           // 二级分类数组（可为空）
        {
          id: Number,       // 二级分类ID（11, 12, 13, ...）
          name: String,
          sort: Number,
          status: Boolean,
        }
      ],
    }
  ],

  // ===== 壁纸数组 =====
  wallpapers: [
    {
      id: Number,           // 唯一标识（自增，从101起步）
      title: String,        // 壁纸标题
      catId: Number,        // 所属分类ID（一级或二级）
      thumb: String,        // 缩略图URL (600×340)
      originalImg: String,  // 原图URL（高清大图，可选）
      authorAvatar: String, // 作者头像URL（UGC专用，可选）
      status: Boolean,      // 是否启用
      sort: Number,         // 排序权重
      date: String,         // 发布日期 (YYYY/MM/DD)
      badge: String,        // 角标文字 (NEW/HOT/精选/空)
      extFields: [          // 扩展字段（动态键值对）
        { key: String, value: String, show: Boolean }
      ],
      resFiles: [           // 多分辨率下载资源
        { label: String, type:'横版'|'竖版', url: String }
      ],
    }
  ],

  // ===== 投稿配置（单一对象）=====
  submitConfig: {
    catId: Number,          // 关联的UGC分类ID
    btnText: String,        // 投稿按钮文案
    email: String,          // 收稿邮箱地址
    desc: String,           // 描述文案
    rules: String,          // 投稿规则（\n换行）
    format: String,         // 允许的图片格式
    minRes: String,         // 最低分辨率要求
    emailTitle: String,     // 邮件标题模板
    status: Boolean,        // 是否启用
  }
}
```

### 5.3 默认数据概要

| 数据项 | 数量 | 说明 |
|---|---|---|
| Banner | 3条 | 版本KV / 精选同人 / 周年庆典 |
| 一级分类 | 2个 | 官方壁纸(normal) + 同人画廊(ugc) |
| 二级分类 | 3个 | 版本KV / 人物立绘 / 活动/节日壁纸 |
| 壁纸总数 | **23张** | 版本KV 5 + 人物立绘 6 + 活动4 + 同人8 |

> 每张壁纸均配有 2~5 个多分辨率下载链接（横版+竖版），部分配有高清原图和作者头像信息。

### 5.4 ID 编号规范

| 实体 | ID 范围 | 说明 |
|---|---|---|
| 一级分类 | 1, 2, ... | 自增 |

## 9. 技术约束 & 注意事项

### 9.1 当前实现方式

- **纯前端实现**：HTML + CSS + JavaScript，无需构建工具，双击即可运行
- **零依赖**：不引入任何第三方库（Vue/React/jQuery 等）
- **数据持久化**：localStorage（同域共享）

### 9.2 已知限制

| 限制 | 影响 | 建议 |
|---|---|---|
| localStorage 5~10MB 上限 | 数据量过大可能溢出 | 正常图文数据足够；如需大量壁纸应迁移至后端API |
| 无用户鉴权 | 任何人可访问 admin.html | 生产环境需加登录验证 |
| 无图片上传 | 只支持填写图片URL | 需对接 CDN/OSS 上传服务 |
| 无真实投稿系统 | 仅展示投稿规则+邮箱引导 | 后续需对接后端审核流程 |
| 下载使用 window.open | 新标签页打开图片而非触发下载 | 上线前改用 `<a download>` 或 fetch+blob |

### 9.3 后续迭代方向（建议）

1. **接入后端 API**：替换 localStorage，支持数据库持久化
2. **图片上传服务**：集成 OSS/COS，支持拖拽上传+裁剪
3. **瀑布流/Masonry 布局**：更接近原神壁纸站的视觉效果
4. **投稿审核工作流**：用户在线提交 → 运营审核 → 自动上架
5. **移动端适配优化**：独立的 H5 适配方案（触摸手势等）
6. **SEO 优化**：SSR 或预渲染支持搜索引擎收录
7. **数据统计埋点**：壁纸下载量、浏览量、搜索热词统计
8. **真实下载**：替换 window.open 为 fetch + blob 下载方式
