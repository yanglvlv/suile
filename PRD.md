## 1. 产品概述

### 1.1 产品定位

DNF 官方壁纸站是地下城与勇士（DNF）游戏官方的**高清壁纸展示与分发平台**，采用前后台分离的单页应用架构（index.html 前台 + admin.html 后台），数据通过 localStorage 同域共享。面向全平台用户提供：

- **官方壁纸**：版本KV主视觉、活动/节日限定壁纸，运营上传多分辨率图片供玩家预览与下载
- **同人作品**：社区创作者投稿的优质同人作品展示，配套公告/活动信息和投稿入口

## 2. 前台功能（index.html）

### 2.1 导航栏

**位置与样式：**
- 固定在页面顶部（`position:fixed`），高度 60px
- 背景为半透明白色（`rgba(255,255,255,0.92)`）+ 毛玻璃模糊效果（`backdrop-filter:blur(20px)`）
- 底部有 1px 浅色描边 + 微阴影，使其与下方内容产生层次分离

**三栏布局（最大宽度 1320px 居中）：**

| 区域 | 内容 | 说明 |
|---|---|---|
| **左侧** | Logo 文字 | 主标题「DNF 壁纸站」使用蓝紫渐变色，副标题「WALLPAPER GALLERY」灰色小字 |
| **中间** | 一级分类导航按钮 | 根据 `DATA.categories` 动态渲染。每个按钮带对应图标：官方壁纸用图片图标 📁，同人作品用星星图标 ⭐。当前激活项为蓝色文字+底部蓝色短下划线指示器（宽20px，高2.5px，圆角），非激活项为灰色。点击切换一级分类 |
| **右侧** | 搜索框 | 圆角胶囊型输入框（`border-radius:22px`），左侧放大镜图标。placeholder 为"搜索版本名称、作者..."，按 Enter 键触发全局搜索 |

**默认分类结构：**
```
官方壁纸 (type='normal', id=1)
  ├── 版本KV (id=11)
  └── 活动/节日壁纸 (id=13)
同人作品 (type='ugc', id=2)
  └── 无子分类，壁纸直接挂在一级分类下
```

### 2.2 Banner 走马灯轮播

**位置：** 导航栏下方，全宽区域，内容区最大宽度 1320px 居中

**布局结构：**
- 同一时刻展示 3 张 Banner 图：中间为当前焦点大图（占容器 60% 宽度，16:9 比例），左右各露出一张缩略小图
- 左右小图缩放到 82%（`transform:scale(0.82)`），半透明（`opacity:0.5`）+ 暗化（`filter:brightness(0.7)`），形成景深透视效果
- 其余 Banner 设为 `opacity:0` 且 `pointer-events:none` 隐藏

**交互与动效：**

| 功能 | 说明 |
|---|---|
| **切换动画** | 所有 slide 的 `left`、`transform`、`opacity` 属性使用 `transition: all .5s cubic-bezier(.4,0,.2,1)` 过渡 |
| **自动播放** | 每 5 秒自动切换到下一张（`setInterval`），用户手动操作后重置计时器 |
| **底部指示器** | 圆点组，当前项宽 36px 蓝色长条，非当前项宽 20px 灰色短条。点击圆点跳转对应 Banner |
| **左右箭头** | 指示器两侧的圆形按钮（36px），点击切换上一张/下一张。白色半透明背景 + 边框，hover 变蓝 |
| **点击跳转** | 若 Banner 配置了 `link` 字段，点击中间大图跳转对应链接 |

**Banner 数据字段：** `{ id, img, tag, link, sort, status }`
- `tag`：仅后台管理用的内部标签（如"最新版本"），**不在前端展示文字**
- `img`：由运营提供完整设计好的图片（含文字排版），系统只负责展示图片

### 2.3 二级分类 Tab 栏

**显示条件：** 仅当当前一级分类配置了子分类（`children` 数组非空）时显示

**位置：** Banner 下方，最大宽度 1320px 居中，左对齐

**结构：**
- 最左边固定一个"全部"按钮（选中 `activeSubCatId=null` 时聚合所有子分类下的壁纸）
- 右侧依次排列各子分类 Tab 按钮

**样式：**
- 胶囊形按钮（`border-radius:20px`，内边距 `7px 20px`）
- 激活态：白色文字 + 蓝色背景（`#3b7ddd`），`font-weight:600`
- 非激活态：灰色文字，hover 变浅蓝背景
- 各按钮间距 6px

**无子分类的一级分类（如同人作品）：** Tab 栏区域不渲染，壁纸直接挂在一级分类 id 下

### 2.4 壁纸列表（双布局模式）

壁纸列表容器 `.wallpaper-grid` 位于 Tab 栏下方，最大宽度 1320px 居中。根据当前分类类型（`category.type`）自动切换布局方式。

#### 2.4.1 官方壁纸布局（type='normal'）

采用分组 Grid 布局，营造大小交错的视觉节奏感：

**分组规则：**
- 每 **5 张壁纸**为一组，渲染为一个 `.grid-row`（`display:grid; grid-template-columns:repeat(4,1fr); grid-template-rows:1fr 1fr; gap:10px; height:280px`）
- 每组第 **1 张为大图**（`.wp-card-big`），占 2 列 2 行（即左半边或右半边的完整高度）
- 每组第 **2~5 张为小图**，各占 1 格

**大图左右交替：**
- 第 1 组（`groupNum=0`）：大图在**左侧**（grid-column 1~2），4 张小图在右侧 2×2
- 第 2 组（`groupNum=1`）：大图在**右侧**（grid-column 3~4），4 张小图在左侧 2×2
- 以此交替，通过 `.grid-row-right` 类控制大图和小图的网格位置

**尾部不足 5 张：** 使用 `.grid-row-flat` 类（`grid-template-rows:1fr; height:150px`），所有卡片等大 4 列单行排列

**分页：**
- `PAGE_SIZE = 20`（每页 20 条壁纸，即 4 个完整分组行）
- 底部渲染分页器：上一页/下一页按钮 + 页码按钮（超过 7 页时中间用 `…` 省略）+ 当前范围提示（如 `1-20 / 36`）
- 点击页码后平滑滚动到壁纸列表顶部（`scrollIntoView`）

#### 2.4.2 同人作品布局（type='ugc'）

采用 CSS 多列瀑布流布局，自然呈现横竖混排的错落美感：

**瀑布流实现：**
- 容器添加 `.masonry` 类：`column-count:4; column-gap:12px`
- 每张卡片设置 `break-inside:avoid; margin-bottom:12px; display:inline-block; width:100%`
- 卡片内图片使用 `width:100%; height:auto` 保持原始比例，竖版图设 `max-height:360px; object-fit:cover` 防止过长

**横竖交替排序逻辑：**
- 壁纸数据含 `orient` 字段：`'h'` 表示横版（16:9），`'v'` 表示竖版（如 9:16）
- 渲染前将横版和竖版分别收集，然后按"2 横 + 1 竖 + 1 横"的节奏交替插入，使瀑布流中横竖图均匀分布，避免某一列全是竖图导致高度失衡

**不分页：**
- 同人作品区不使用分页，一次性展示所有壁纸（当前 36 张）
- 底部显示"海量作品"提示卡片（蓝紫渐变背景 + 装饰圆形元素），内容为"✨ 海量作品 持续更新中"+"共收录 XX 幅同人佳作 · 更多创作者作品正在路上"

#### 2.4.3 搜索结果布局

**触发方式：** 用户在搜索框输入关键词后按 Enter 键

**搜索范围：** 跨所有分类的全部壁纸（`DATA.wallpapers`），匹配以下字段：
- `title`（壁纸标题）
- `date`（发布日期）
- `extFields[].value`（扩展字段值，如版本名称、创作者等）

均为大小写不敏感的子串匹配（`toLowerCase().includes()`）

**结果展示：**
- 搜索结果按**一级分类分组**显示，每组有独立的分类标题头：
  - 标题头包含：分类图标（📁 官方壁纸 / ⭐ 同人作品）+ 分类名称（15px 加粗）+ 右侧结果计数标签（灰色背景圆角小标签，如"2 条结果"）
  - 标题头底部有 2px 灰色分割线
- 每组内的壁纸采用**统一的 4 列等高 Grid 平铺**布局（`grid-template-columns:repeat(4,1fr)`，卡片高度 160px），不区分大小图，确保搜索结果整齐清晰
- 搜索结果不分页，全部展示
- 顶部数据计数显示"共 X 条数据"

**退出搜索：** 清空搜索框后按 Enter，或点击导航栏的分类按钮，都会退出搜索模式回到正常浏览

#### 2.4.4 通用壁纸卡片结构

每张壁纸卡片（`.wp-card`）的内部结构：

| 层级 | 元素 | 说明 |
|---|---|---|
| 最外层 | `.wp-card` | 圆角 10px，溢出隐藏，深色背景（`#1a1a2e`），入场动画 `fadeUp`（从下方淡入上移 16px） |
| 角标 | `.wp-badge` | 左上角绝对定位，蓝色背景白色文字圆角标签，显示 `badge` 字段内容（如 NEW、HOT、精选） |
| 图片 | `.wp-card-img > img` | 填满卡片，`object-fit:cover`，`loading="lazy"` 懒加载，hover 时 `transform:scale(1.05)` 微放大 |
| 信息遮罩 | `.wp-card-overlay` | 底部渐变黑色遮罩（`transparent → rgba(0,0,0,0.7)`），包含标题（白色12px加粗，单行省略）、日期+元数据（白色半透明10px）、作者信息（仅同人作品显示，圆形头像14px + 作者名） |

**Hover 效果：** 卡片整体上浮 2px + 增强阴影（`box-shadow:0 8px 28px rgba(0,0,0,0.15)`）

### 2.5 投稿入口横幅（仅同人作品分类）

**显示条件：** 当前一级分类的 `type === 'ugc'` 且 `submitConfig.status === true` 且 `submitConfig.catId === activeCatId`

**位置：** 二级 Tab 栏与公告活动区之间

**结构：**
- 浅蓝紫渐变背景卡片（`linear-gradient(135deg, #eef4ff, #f0ecff)`），蓝色描边，圆角 14px
- 左侧：标题"投稿你的作品"（15px 加粗深蓝色）+ 副标题"欢迎DNF创作者投稿原创同人作品"（12px 灰色）
- 右侧：蓝色"我要投稿"按钮（胶囊形，`border-radius:22px`），hover 变深蓝 + 阴影

**点击"我要投稿"按钮：** 弹出投稿说明弹窗（全屏遮罩 + 白色内容卡片 460px 宽），显示投稿邮箱和投稿规则说明文本

### 2.6 公告 + 活动区（仅同人作品分类）

**显示条件：** 与投稿横幅相同

**位置：** 投稿横幅下方、壁纸列表上方

**布局：** 两栏等宽并排（`grid-template-columns:1fr 1fr; gap:16px`）

| | 左栏：公告 | 右栏：活动 |
|---|---|---|
| **卡片背景** | 白色，圆角 12px，1px 描边 | 同左 |
| **标题栏** | 蓝色铃铛 SVG 图标 + "公告" + 右侧"更多 →"按钮 | 橙色星星 SVG 图标 + "活动" + 右侧"更多 →"按钮 |
| **列表项** | 取前 4 条（按 `sort` 字段升序），每条：蓝色圆点 + 标题文本（含可选标签如 NEW）+ 右侧日期 | 同左，但圆点为橙色 |
| **点击行为** | 每条点击跳转 `location.hash='#/notice/{id}'` | 每条点击跳转 `location.hash='#/activity/{id}'` |
| **"更多"按钮** | 跳转 `#/notice` 列表页 | 跳转 `#/activity` 列表页 |

**标签样式：** `new` 标签为浅蓝背景蓝色文字，`hot` 标签为浅橙背景橙色文字，字体 10px 加粗

### 2.7 Hash 路由页面（公告/活动的列表页与详情页）

路由切换时的页面结构变化：
1. 隐藏主页所有区块（Banner、Tab 栏、投稿横幅、公告活动区、壁纸列表）
2. 显示路由页面容器 `#routePage`（最大宽度 900px 居中，上边距 80px 避开导航栏）
3. 页面滚动到顶部

**路由匹配规则（正则 `^#\/(notice|activity)(?:\/(\d+))?$`）：**

| 路由 | 渲染内容 |
|---|---|
| `#/notice` | 公告列表页 |
| `#/notice/{id}` | 公告详情页 |
| `#/activity` | 活动列表页 |
| `#/activity/{id}` | 活动详情页 |
| 其他（含空 hash） | 回到主页 |

**列表页结构：**
- 顶部"← 返回壁纸站"按钮（点击 `location.hash=''`）
- 页面标题：📢 公告列表 / ⭐ 活动列表
- 列表项：白色卡片（圆角 12px，1px 描边），每项包含彩色圆点 + 标题（含可选标签）+ 日期 + 右箭头 `›`，hover 向右位移 4px + 浅蓝阴影。点击跳转对应详情页

**详情页结构：**
- 顶部"← 返回"按钮（使用 `history.back()` 导航，保证从列表页进入时能正确返回列表而非主页）
- 标题（20px 加粗）+ 元信息行（日期 + 可选标签）
- 正文区：白色卡片（圆角 14px，内边距 28px 32px），`font-size:14px`，`line-height:2`，支持渲染 HTML 富文本内容（加粗、斜体、标题、列表、引用块、图片等）。若内容为纯文本（不含 HTML 标签），自动将 `\n` 转为 `<br>` 换行

### 2.8 壁纸预览弹窗

**打开方式：** 点击任意壁纸卡片触发 `openPreview(idx)`

**弹窗结构：** 全屏半透明遮罩（`rgba(0,0,0,0.6)` + `backdrop-filter:blur(4px)`），内容区 `flex` 水平排列（最大宽度 1140px，92% 视口宽度，最大高度 88vh）

| 区域 | 内容 |
|---|---|
| **左侧预览区** | 壁纸大图（`max-height:82vh; object-fit:contain`，圆角 10px + 深阴影）。默认加载 `originalImg` 字段（若有），否则将缩略图 URL 中的尺寸替换为 1200×680 的放大版 |
| **左右切换箭头** | 预览区左右两侧的圆形半透明按钮（40px），点击切换到当前列表中的上/下一张壁纸 |
| **底部缩放条** | 半透明黑色条（圆角 20px），包含 `−` 按钮 + 当前缩放百分比 + `+` 按钮。范围 25%~300%，步进 25%，通过 `transform:scale()` 实现 |
| **右侧信息面板** | 白色卡片（宽 300px，圆角 14px），顶部壁纸标题（18px 加粗）+ 日期 + 分类路径 + 扩展字段列表。底部"选择壁纸尺寸"下载区 |
| **右上角关闭** | `×` 按钮，圆形半透明白色背景 |

**下载区交互（关键设计）：**
- 多分辨率图片按横版/竖版分组展示，每组有"横版尺寸"/"竖版尺寸"分组标题
- 每个分辨率项为一行，分两个可点击区域：
  - **左侧文字区**（分辨率标签如"1920×1080"）：点击后将预览区大图**切换为该分辨率的图片**，方便用户预览不同尺寸效果
  - **右侧下载图标**：点击触发浏览器下载（创建 `<a download>` 元素）
- 两个区域用竖线分隔，hover 分别有独立的视觉反馈（文字区变浅蓝背景，图标区变蓝色背景白色图标）

**键盘快捷键（弹窗打开时）：**

| 按键 | 功能 |
|---|---|
| `←` | 切换到上一张壁纸 |
| `→` | 切换到下一张壁纸 |
| `+` 或 `=` | 放大预览图（步进 25%） |

## 4. 数据模型

### 4.1 存储方案

| 项目 | 说明 |
|---|---|
| **存储介质** | 浏览器 `localStorage` |
| **存储键名** | `dnf_wallpaper_config` |
| **共享机制** | index.html 和 admin.html 同域部署，读写同一个 localStorage 键，数据实时共享 |
| **版本控制** | 数据根对象包含 `_ver` 字段，前后台各自定义 `DATA_VERSION` 常量（当前为 `'v2.2'`）。页面加载时比较 `_ver` 与 `DATA_VERSION`，不匹配则丢弃旧数据并重置为内置默认数据集 |

### 4.2 完整数据结构

```javascript
{
  _ver: 'v2.2',  // 数据版本号，前后台 DATA_VERSION 常量必须保持一致
  banners: [{
    id: Number,           // 唯一标识
    img: String,          // 图片 URL
    tag: String,          // 内部管理标签（不在前端展示）
    link: String,         // 点击跳转链接（可选）
    sort: Number,         // 排序权重，数字越小越靠前
    status: Boolean,      // true=上线, false=下线
  }],
  categories: [{
    id: Number,           // 一级分类 ID
    name: String,         // 分类名称
    type: String,         // 'normal'=普通类 | 'ugc'=UGC投稿类
    sort: Number,         // 排序权重
    status: Boolean,      // true=显示, false=隐藏
    children: [{          // 二级分类数组（可为空）
      id: Number,         // 二级分类 ID
      name: String,
      sort: Number,
      status: Boolean,
    }],
  }],
  wallpapers: [{
    id: Number,           // 唯一标识
    title: String,        // 壁纸标题
    catId: Number,        // 所属分类 ID（指向二级分类 ID，无子分类时指向一级分类 ID）
    thumb: String,        // 缩略图 URL（列表页展示用）
    orient: String,       // 图片方向：'h'=横版, 'v'=竖版
    originalImg: String,  // 原图高清 URL（预览弹窗用，可选）
    authorAvatar: String, // 作者头像 URL（同人作品专用，可选）
    status: Boolean,      // true=上线, false=下线
    sort: Number,         // 排序权重
    date: String,         // 发布日期，格式 'YYYY/MM/DD'
    badge: String,        // 角标文字（如 'NEW'、'HOT'、'精选'，可选）
    extFields: [{         // 扩展信息字段数组
      key: String,        // 字段名（如"版本名称""创作者"）
      value: String,      // 字段值
      show: Boolean,      // 是否在前端展示
    }],
    resFiles: [{          // 多分辨率图片数组
      label: String,      // 分辨率标签（如"1920×1080"）
      type: String,       // '横版' | '竖版'
      url: String,        // 该分辨率的图片 URL
    }],
  }],
  notices: [{
    id: Number,           // 唯一标识
    text: String,         // 公告标题
    date: String,         // 日期，格式 'YYYY/MM/DD'
    tag: String,          // 标签（'new' 或空）
    sort: Number,         // 排序权重
    status: Boolean,      // true=上线, false=下线
    content: String,      // 详情内容（HTML 格式或纯文本）
  }],
  activities: [{
    id: Number,           // 唯一标识
    text: String,         // 活动标题
    date: String,         // 日期
    tag: String,          // 标签（'hot'、'new' 或空）
    sort: Number,         // 排序权重
    status: Boolean,      // true=上线, false=下线
    content: String,      // 详情内容（HTML 格式或纯文本）
  }],
  submitConfig: {
    catId: Number,        // 关联的 UGC 类型一级分类 ID
    email: String,        // 投稿收件邮箱
    content: String,      // 投稿说明全文
    status: Boolean,      // true=启用, false=禁用
  },
}
```
